The problem

Your agent runs
while you're away.

You start Claude Code on a long task and walk away. It hits an approval prompt and stalls for an hour. Or worse — it doesn't stall, it runs rm -rf while you're at lunch. Every new session starts amnesiac, repeating mistakes it already made yesterday.

pocket-t puts the session in your pocket. Read output as chat. Approve or deny tool calls from your lock screen, before they execute. And Memento gives the agent a memory that survives across sessions — deterministic, no LLM, zero added latency.

What it does

Watch. Approve.
Remember. Integrate.

Watch

Terminal → phone

Every session becomes a chat contact. Read output, type back, scroll history with push notifications. Desktop gets a full xterm.js terminal.

Approve

One-tap, pre-execution

Real blocking approval for agent tool calls — not a notification after the fact. Allow or deny destructive commands from your lock screen.

Remember

Memento memory

Constraints, failures, and decisions from past sessions distilled into NOHUP.md, injected before every tool call. Zero LLM cost.

Integrate

MCP-ready

MCP — pocket-t as agent tooling

Native tools for any
MCP-compatible host.

Sessionslist_sessions · get_session_output · send_input · spawn_session · kill_session
Any MCP host can enumerate every terminal session on your Mac, read recent output, type input as if it were you, spawn a fresh CLI, or kill a runaway. The supervision layer, exposed as tools.
Approvalsapprove_tool_call
Resolve a pending blocking approval from another agent or a script. Pair with list_sessions filtered to status waiting to build approval automations.
Memory (Memento)get_context · get_constraints · get_patterns · get_decisions · query_brain · log_event
Run pocket-t serve-mcp and any host can read the project's compiled memory: locked user constraints, known-failure patterns, architectural decisions, or a keyword search across the brain. Read it at session start; it costs zero LLM tokens.
CLIrun · --memento · mcp · serve-mcp · dream · init · scan · skill
A single binary. run starts the daemon, --memento turns on memory, mcp/serve-mcp expose MCP servers, dream runs the nightly consolidation cycle, init scopes a project, skill prints a Claude-Code skill doc.
Memento — the memory layer

The memory system from
the film, for AI agents.

Deterministic. No LLM. Memento watches session output, stamps salience at encoding time, and maintains a live compressed NOHUP.md the agent reads before it acts — the tattoo it can't forget.

Corroboration gate, Ebbinghaus decay curve, in-place reconsolidation on correction, Leitner spaced-repetition, and a nightly dream consolidation pass that extracts cross-session schemas.

Agents retrying failed approaches
Known-failure patterns promoted after corroboration, surfaced before the retry.
Constraints lost mid-session
User rules are locked and immune to decay — said once is a rule forever.
Amnesiac session starts
NOHUP.md injected via the PreToolUse hook, every session, automatically.
Quota burned on retry loops
Zero LLM, zero latency — a file read on the hot path. Nothing else ships this.
Architecture

Outbound by design.
No inbound ports, ever.

  Your Mac                       Relay (you own it)          Your phone
  +--------------------+         +-----------------+         +--------------+
  | pocket-t daemon    | outbound| Fastify+SocketIO| outbound| React PWA    |
  | PTY . ANSI . hooks |  WSS -->| routes,persists |<-- WSS  | chat,approve |
  | Memento . MCP      |         | never initiates |         | xterm . push |
  +--------------------+         +-----------------+         +--------------+

Both endpoints connect out. The relay never initiates a connection — so pocket-t works through NAT, hotel WiFi, and LTE with zero network config. E2E encryption is optional, AES-128-GCM, and fully open source: the relay routes ciphertext it cannot read.
